5. REMAINING ISSUES (PRIORITY ORDER)
================================================================================

PRIORITY 1 — AI Perspective Enforcement
  AI scenarios still use 3rd person 75% of the time ("What should the
  pitcher do?" instead of "You're the pitcher..."). The prompt fix was
  partially effective. Consider adding a post-generation regex replacement
  that rewrites 3rd person to 2nd person before display.
  SEVERITY: Medium — doesn't break functionality but weakens immersion

PRIORITY 2 — AI Pool Depth
  Only 3 of 7 positions had pre-cached AI scenarios. Pool fill rate needs
  to be more aggressive, especially for popular positions. Consider:
  - Pre-fetching 2-3 scenarios per position on app load
  - Server pool (D1) to serve as a deeper fallback before handcrafted
  - Monitoring pool depth per position
  SEVERITY: Medium — fallback UX is now excellent, so impact is lower

PRIORITY 3 — Live Generation Speed
  When pools are empty and live generation runs, it still takes 25-60+
  seconds. This is mostly an xAI API latency issue but could be improved
  with faster model selection or prompt optimization.
  SEVERITY: Low — pool hits are fast, fallbacks are fast, this is the
  uncommon path now

================================================================================
7. SUMMARY
================================================================================

The 14 fix prompts transformed BSM's AI experience from a C+ to a B+. The
three biggest wins were:

1. COACH LINES (D → A-): From zero educational value to 100% context-aware
   teaching moments with real baseball data. This alone changed the feel of
   every scenario interaction.

2. FALLBACK UX (C → A): From silent/confusing failures to a polished
   experience with clear communication, appropriate difficulty, and seamless
   transitions. The fallback path is now genuinely good.

3. DEDUPLICATION (F → A): From serving identical scenarios back-to-back to
   zero observed duplicates. The completed-ID check is working.

The main remaining weakness is AI delivery rate (D+). The pipeline itself
works, but pool depth is insufficient — most positions have no pre-cached
scenarios, forcing slow live generation or handcrafted fallbacks. However,
this is now a PERFORMANCE issue, not a QUALITY issue. When AI delivers, it's
excellent. When it falls back, the experience is smooth and educational.

The path from B+ to A is clear:
1. Fix AI perspective to 2nd person (B- → A would add ~0.12 to weighted)
2. Deepen pre-cache pools (D+ → B would add ~0.23 to weighted)
3. These two fixes alone would push the grade to ~3.72 (A-)
